Troubleshooting

This page lists common support questions along with answers. You can also have a look at the list of known issues.

VSTU's solution won't load in Visual Studio

Please see the answer to this stackoverflow question.

On Windows 8, opening the UnityVS solution in Visual Studio results in an error window asking to download the Unity target framework

UnityVS requires that you install the .net framework 3.5 on Windows 8.

UnityVS 1.8 to VSTU 1.9 migration

If you are migrating an existing UnityVS 1.8 solution, we recommend that you:

  • Delete the UnityVS .sln and .*proj files from the root folder of your Unity project.
  • Import the new Visual Studio Tools for Unity package into your Unity project as shown in the documentation.
Your new solution will be automatically generated.
