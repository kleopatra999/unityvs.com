Known issues

  • When being debugged, Unity's scripting engine is known to crash occasionally. Unity uses an old version of their scripting engine.
  • When being debugged, Unity can freeze. This is also related to the fact that their scripting engine is an old version which is prone to deadlocks. If this freezes Visual Studiot too, please apply
  • Stepping in and out of methods is sometimes capricious, especially in iterators or within switch statements. Again, this is related to the fact that Unity's C# compiler is an old version.

For all issues that can not be addressed or worked around from VSTU, we're collaborating upstream to ensure they're fixed.

Please report all crashes. Thanks!

What to do when Visual Studio locks up?

We've had reports that Visual Studio sometimes locks up when debugging. We need data to help fix those. Here's what you can do to help us whenever Visual Studio is locked up.

  • Open another Visual Studio instance,
  • In the new Visual Studio instance, do Tools -> Attach to Process,
  • Select and attach the debugger to the Visual Studio instance that is locked up,
  • Pause the debugger,
  • Go to the Commands window pane, if you don't have go to View -> Other Windows -> Command Window,
  • In the Command window pane input the command: 
    Debug.ListCallStack /AllThreads /ShowExternalCode
  • Send us the whole thread dump, along with a description of when the lock up at vstusp@microsoft.com,

This will help us investigate and fix those issues.
